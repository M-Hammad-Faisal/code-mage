Before you write your first test, your machine needs to be set up correctly. Skipping this chapter is how people end up stuck on installation errors for hours.

This chapter assumes you have never touched test automation before. If you already have Node.js 18+ installed and you're comfortable in a terminal, skip to the next chapter.

## What You Actually Need

- **Node.js** (version 18 or later) — the runtime that executes JavaScript/TypeScript outside a browser
- **npm** — the package manager that comes bundled with Node.js
- **VS Code** — the editor we'll use throughout this tutorial
- **A terminal** — the black window where you type commands

That's it. No other tools. No paid software.

## Installing Node.js

Go to [https://nodejs.org](https://nodejs.org) and download the **LTS** version (Long Term Support). Do not download the "Current" version — it changes too frequently and some packages don't support it yet.

Run the installer. It installs both Node.js and npm automatically.

Verify the installation by opening your terminal and running:

```bash
node --version
npm --version
```

You should see something like:

```
v20.11.0
10.2.4
```

If you get `command not found`, close and reopen your terminal and try again. If it still fails, restart your machine — the installer sometimes requires it.

### What is Node.js?

You've written code that runs in a browser before — clicking a button, showing an alert. Node.js lets you run JavaScript on your machine, outside any browser. Playwright uses Node.js to control browsers, make network requests, and run your test files.

You don't need to be a Node.js expert. You just need it installed.

## Installing VS Code

Go to [https://code.visualstudio.com](https://code.visualstudio.com) and download the version for your OS.

Install these extensions (open VS Code → press `Ctrl+Shift+X` or `Cmd+Shift+X` on Mac → search by name):

- **Playwright Test for VS Code** (by Microsoft) — run and debug tests directly from the editor
- **ESLint** — highlights code errors as you type
- **Prettier** — formats your code automatically
- **TypeScript** — comes built-in, but make sure it's enabled

The Playwright VS Code extension is genuinely useful — you can click a play button next to any test to run it, and it highlights which line failed.

## Understanding the Terminal

The terminal (also called the command line, shell, or console) is where you run commands. On Windows it's called Command Prompt or PowerShell. On Mac and Linux it's called Terminal.

You'll use it constantly. Here are the commands you need for this tutorial:

```bash
# Print the current folder you're in
pwd

# List files in the current folder
ls          # Mac/Linux
dir         # Windows

# Move into a folder
cd folder-name

# Move up one folder
cd ..

# Create a new folder
mkdir folder-name

# Run a Node.js script
node script.js

# Install packages
npm install

# Run an npm script defined in package.json
npm run test
```

You don't need to memorize these. Just know they exist and refer back here when needed.

## Setting Up Your Project Folder

Create a folder for this tutorial project. Open your terminal and run:

```bash
mkdir playwright-saucedemo
cd playwright-saucedemo
```

Now open it in VS Code:

```bash
code .
```

If `code .` doesn't work on Mac, open VS Code manually → press `Cmd+Shift+P` → type "Shell Command: Install 'code' command in PATH" → press Enter → try again.

You should now have VS Code open with an empty project folder. The terminal at the bottom of VS Code is the same as your system terminal — you can use either one.

## Understanding npm and package.json

When you install Playwright, it gets added to a file called `package.json`. This file tracks every package your project depends on, along with scripts you can run.

```json
{
  "name": "playwright-saucedemo",
  "version": "1.0.0",
  "scripts": {
    "test": "playwright test"
  },
  "devDependencies": {
    "@playwright/test": "^1.40.0"
  }
}
```

When you run `npm install`, npm reads `package.json` and downloads all listed packages into a `node_modules` folder.

**Never commit `node_modules` to Git.** It can contain thousands of files. Instead, commit `package.json` and `package-lock.json`. Anyone who clones your repo runs `npm install` and gets the same packages.

Create a `.gitignore` file in your project root:

```
node_modules/
playwright-report/
test-results/
.playwright/
playwright/.auth/
```

## What is a Package?

A package is code someone else wrote that you can use in your project. Playwright is a package. You install it with:

```bash
npm install @playwright/test
```

The `@playwright/test` is the package name. The `@playwright/` prefix means it's published under the "playwright" organization on npm.

When you install a package, it gets added to `node_modules` and listed in `package.json`. You import it in your code with:

```typescript
import { test, expect } from '@playwright/test';
```

This tells Node.js to look in `node_modules/@playwright/test` and use the code from there.

## A Note on File Paths

Throughout this tutorial, paths are written relative to your project root — the `playwright-saucedemo` folder.

So when you see `tests/auth/login.spec.ts`, it means create a file called `login.spec.ts` inside a folder called `auth` inside a folder called `tests`, all inside your project root.

```
playwright-saucedemo/       ← project root
  tests/
    auth/
      login.spec.ts         ← this file
  pages/
  package.json
```
